# Domain B: Contributor & Gamification Domain Discussion History

본 문서는 일반 비장애인 기여자의 매장 등록 편의성 증진 및 크롬 공룡 게이미피케이션 동기부여 인프라 구축에 관한 핵심 아이디어와 의사결정 히스토리입니다.

---

## 1. 수동 타이핑 오염 방지를 위한 Google Maps SDK 도입 (2026-07-15)
* **논의 배경:** 기여자가 매장 이름을 수동으로 입력할 시 "맥도날드 홍대", "맥도날드 홍대점", "멕도날드" 등 오타와 정합성 왜곡이 발생해 시각장애인 GPS 매치와 Firestore NoSQL 락인 시 검색 매칭률이 오염되는 치명적 문제가 발생함.
* **의사결정:** 구글 맵스 API 및 Places SDK를 결합하여 정확한 장소 식별자(Google Place ID)를 획득하고 이를 Firestore의 식별 키로 사용하도록 인프라를 강제했습니다.

---

## 2. GPS 내 위치 추천 및 Autocomplete 하이브리드 기여 (2026-07-15)
* **논의 배경:** 기여를 쉽게 만들기 위해 현장에 직접 서 있는 기여자는 물론, 집이나 외부에서 전국 매장 정보를 미리 기여하려는 기여자를 동시에 수용해야 함.
* **의사결정:**
  - **📍 내 주변 매장 기여:** Geolocation API를 호출해 내 반경 50m 주변의 매장을 드롭다운으로 즉시 선택.
  - **🔍 전국 매장 기여:** Google Maps Autocomplete API 검색창을 배치하여 텍스트 일부 입력 시 전국 장소 자동완성 선택지 노출.
  - 선택 완료 시, 프론트에 미니 구글 맵 핀 마커(`renderMiniGoogleMap`)를 실시간 렌더링하여 정확성을 교차 검증시켰습니다.

## 3. 크롬 픽셀 공룡(Dino) 레벨 업 이스터에그 게이미피케이션 (2026-07-15)
* **논의 배경:** 단순 자원봉사 방식으로는 지속적인 매장 데이터 수집에 필요한 트래픽 동력을 얻을 수 없음.
* **의사결정:** 구글 크롬의 아이코닉한 이스터에그 캐릭터인 픽셀 공룡(Dino)을 적극 채용하여, 기여 점수에 따라 알(🥚) ➔ 아기 공룡(🦕) ➔ 달리는 공룡(🦖) ➔ 히어로 디노(👑🦖)로 100레벨 진화하게 만들고 명예의 전당 리더보드에 카드를 가시화하여 구글 심사위원들의 유머 코드와 동기부여를 동시에 잡도록 기획했습니다.
