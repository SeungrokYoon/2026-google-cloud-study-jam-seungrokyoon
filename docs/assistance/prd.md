# Domain A: Assistance Domain PRD (Product Requirement Document)

본 문서는 시각장애인 음식 주문 및 키오스크 조향 도메인의 최종 기능 요구사항(PRD) 명세서입니다.

---

## 1. 글로벌 다국어 온보딩 (Sequential Multi-Language Onboarding)
* **요구사항 1:** 영구 저장된 언어가 없을 시, 7개국어(한국어/영어/중국어/일본어/러시아어/독일어/아랍어)로 시작 안내를 3.5초 주기로 교차 낭독해야 한다.
* **요구사항 2:** 낭독 중 더블 탭 시 해당 임시 언어로 전환되며 *"이 언어로 설정하시겠습니까? 저장하려면 더블 탭, 취소하려면 길게 누르세요"* 가이드 컨펌 절차를 제공한다.
* **요구사항 3:** 저장 수락 시 `localStorage.setItem("user_language")`에 영구 락인하여 다음 진입 시 오토 플레이되도록 한다.

---

## 2. 배리어프리 권한 온보딩 (Permission Onboarding Bypass)
* **요구사항 1:** 브라우저의 오디오 Autoplay 차단을 회피하기 위해, 최초 더블 탭 유저 인터랙션 내부에서 `AudioContext.resume()`을 강제 호출하여 오디오 노드를 활성화해야 한다.
* **요구사항 2:** 카메라 및 마이크 권한 요청(`getUserMedia`) 팝업 트리거와 동시에 *"화면 상단 중앙의 허용 버튼을 눌러주세요"* 음성 코치를 출력하여 권한 획득 절차를 보청(Aim-Assist Permission)해야 한다.

---

## 3. 대화형 매장 특정 (Dialogue Venue Match)
* **요구사항 1:** Geolocation GPS 좌표를 획득하여 `/api/venue-assist` API를 경유, 주변 100m 이내 매장 리스트 정보를 가져와 오디오 브리핑한다.
* **요구사항 2:** 사용자의 모호한 음성 입력("스타벅스", "맥날")이 감지되면 백엔드에서 Gemini 2.5 Flash를 매칭하여 실제 데이터베이스 Key("맥도날드 홍대점")를 락인(Disambiguation)한다.

---

## 4. 4중 멀티터치 제스처 터치패드 제어 (Zero-UI Gesture Engine)
사용자의 스크린 접촉 손가락 개수를 판정하여 동작을 4중 분기한다.
1. **5지/주먹 대기 모드 (Pause):** 주먹을 쥐고 대거나 스크린을 감싸 쥐는 넓은 면적 접촉 시, 조향 비프음을 끄고 비디오 분석 루프를 동결(`isFreezed = true`)한다. (다시 탭하면 재개)
2. **3지 뒤로가기 (Back Navigation):** 엄지, 검지, 중지의 3점 터치 감지 시, 조향음을 차단하고 *"이전 화면으로 돌아갑니다."* 안내 후 키오스크 화면 단계를 이전 상태로 롤백 렌더링한다. (2초 디바운스 적용)
3. **2지 화면 낭독 탐색 (Explore):** 두 손가락으로 드래그할 시 조향음을 차단하고, 손끝 아래에 겹치는 메뉴 명칭과 정보를 1.5초 간격으로 실시간 TTS 브리핑한다.
4. **1지 조향 유도 및 선택 결정 (Steering & Select):** 한 손가락 접촉 시 3D 오디오 조향 비프음을 개시하고, 타겟 버튼 안착 시 1초 연속 진동 락인을 울리며, 정합 성공 상태에서 1초 대기 시 장바구니에 해당 메뉴를 자동 적재한다.

---

## 5. iOS 대응 3D 사운드 & 비주얼 플래싱 (Stereo Panning & Visual Flash)
* **요구사항 1:** iOS 물리 진동 차단 시, Web Audio API를 활용한 입체 음향 가이드를 전개한다. (우측: `pan.value = 1.0`, 좌측: `pan.value = -1.0`, 상단: `800Hz` 고음, 하단: `250Hz` 저음).
* **요구사항 2:** 저시력 유저의 각도 이탈 보정을 돕기 위해, 화면 사방 테두리에 고대비 글로우 테두리 플래시(`haptic-pulse-left/right/up/down`) 클래스를 부착해 글로잉 애니메이션을 실시간 분출한다.

---

## 6. 조향 도메인 인공지능 모델 믹스 전략 (AI Model Mix Strategy for Assistance)
초저지연 실시간 반응성과 정밀한 오디오 안내를 위해, 구글의 모델 티어(Nano, Flash/Bannano, Pro)를 다음과 같이 하이브리드로 정교하게 믹싱하여 적용한다.

### 6-1. **Gemini 2.5/2.0 Flash (Bannano)** — 초저지연 실시간 비디오OCR 분석 및 애매한 매장 매칭
* **역할:** 사용자가 촬영하는 실시간 비디오 프레임을 OCR 스캔하여 키오스크 화면 상의 버튼 좌표와 텍스트를 실시간 추출하고, 사용자의 모호한 음성 답변("맥날", "벅스")을 실제 GPS 반경 내 매장명("맥도날드 홍대점")과 초고속 퍼지(Fuzzy) 매칭한다.
* **선택 이유:** 시각차단 유저의 손끝 움직임에 실시간 대응하기 위해서는 **TTFT 0.5초 이내의 즉각적인 응답 속도**가 필수적이다. 또한 비디오 프레임을 1초에 여러 차례 백엔드로 스트리밍해야 하므로, **Pro 대비 1/10 가격 경쟁력**을 가진 Flash(Bannano) 모델만이 유일한 현실적 대안이다.

### 6-2. **Gemini 1.5 Pro** — 고정밀 키오스크 메뉴 계층 구조(Hierarchical Menu Tree) 정교화 및 배치 파싱
* **역할:** 일반 기여자가 업로드한 메뉴판 이미지 원본을 기반으로 복잡하게 얽혀 있는 키오스크 메뉴의 상하 계층 구조(예: 단품 선택 ➔ 사이드 변경 ➔ 음료 변경 ➔ 디저트 추가)를 누수 없이 완벽한 JSON 트리 구조로 모델링하고 스키마를 정합한다.
* **선택 이유:** 복잡한 시각 구조 분석 및 엄격한 정형 데이터 JSON 출력(Structured Outputs) 사양에는 **고해상도 멀티모달 인식 및 추론 복잡성**이 뛰어난 `Pro` 등급 모델이 단연 독보적인 성능을 발휘합니다.

### 6-3. **Gemini Nano** — 스마트폰 온디바이스 1차 손 떨림 및 앵글 이탈 스크리닝
* **역할:** 사용자가 키오스크 앞에서 카메라를 조향할 때, 네트워크가 연결되지 않았거나 극심히 지연되는 경우에도 모바일 기기 단독(On-device)으로 카메라 앵글이 키오스크 화면 중앙에서 이탈했는지 여부(Out of Frame)나 극한의 손 떨림(Camera Shake)을 로컬에서 판정하여 즉시 음성 가이드("조금 더 앞으로 가주세요")를 출력한다.
* **선택 이유:** 클라우드 서버 호출로 인한 레이턴시 루프 없이 **0ms 수준의 기기 내 네이티브 하드웨어 반응**을 이끌어내어 배리어프리 실시간 상호작용의 사용성을 비약적으로 높인다.
